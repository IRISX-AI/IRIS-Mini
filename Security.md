# Security Policy

## Supported Versions

Currently, only the latest version of **IRIS-MINI** is supported with security updates. If you are running an older compiled binary or a previous release, please update to the latest version before submitting a security report.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.** Because IRIS-MINI handles API keys, local file system operations, and system-level microphone access, security is our absolute highest priority. If you discover a vulnerability, an exploit, or a bypass mechanism, we ask that you report it privately so we can patch it before it becomes public knowledge.

Please report any security issues directly via:

- **Direct Message on Instagram:** [@201harshs](https://www.instagram.com/201harshs/)
- **Email:** gamerpandeyharsha@gmail.com

**What to include in your report:**

- A description of the vulnerability.
- Step-by-step instructions to reproduce the exploit.
- The operating system and Node.js version you are using.
- Any potential impact on user data or system integrity.

We take all reports seriously and will attempt to acknowledge your report within 48 hours. If the vulnerability is verified, we will issue a patch and credit you in the release notes for the responsible disclosure.
